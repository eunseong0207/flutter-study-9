# 커밋 태그

- [컨벤셔널커밋](https://www.conventionalcommits.org/ko/v1.0.0/)
## 커밋 태그 종류

### 자주 사용되는 것들
- `feat` : 새로운 기능 추가 했을 때
    - ex : `feat : 과제 1번 추가`
- `fix` : 버그 수정 했을 때
    - ex : `fix : 문자열 출력 안되는 현상 수정`
- `docs` : 코드는 건들지 않고 문서를 업데이트 했을때
    - ex : `docs : README 업데이트`
- `refactor` : 일단 작동만 하게 코드를 올려둔걸, 깨끗하게 정리 한 후 남기는 태그
    - ex : `refactor : StudentScore 클래스 리팩토링` 
- `WIP` : Working In Progress. 현재 작업이 완료되지 않았지만 백업용으로 커밋 할 때


### 추가
- `chore` : 귀찮은 일 or 하기 싫은 일 => 패키지 버전 업데이트, 짜잘한 작업(오타수정) , 별거 아닌데 어찌어찌 했다.
- `ci` : 지속적 통합 (CI) 관련된 구성이 수정되었을 때
- `style` : UI 스타일 관련된 파일 수정되었을 때. 주로 프론트엔드에서 css 수정하고 많이 사용함.
- `tyop`: 오타수정
